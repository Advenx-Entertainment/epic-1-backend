1. What is WebSocket?

Normal HTTP works like a walkie-talkie where only one side talks at a time — you send a request, the server replies, and the connection closes. WebSocket is different. The connection stays open permanently and both sides can send messages to each other at any time. This makes it ideal for real-time systems like arena control where the CO needs instant updates from devices.

3. server.py — Line by Line

Imports

- FastAPI — the web framework that runs the server

- Dict — type hint for the connections dictionary

----------------------------------------

ConnectionManager Class
python
class ConnectionManager:
    def __init__(self):
        self.active_connections: Dict[str, WebSocket] = {}

This class is the core of the system. It maintains a dictionary of all currently connected devices. The key is the device ID (a string like "rpi-01") and the value is the WebSocket connection object for that device.

Example of what active_connections looks like at runtime:
python
{
    "co-dashboard": <WebSocket object>,
    "rpi-device-01": <WebSocket object>,
    "pc-node-02": <WebSocket object>
}


----------------------------------------

connect() method
python
async def connect(self, websocket: WebSocket, device_id: str):
    await websocket.accept()
    self.active_connections[device_id] = websocket
    await self.broadcast({
        "type": "status",
        "event": "device_connected",
        ...
    })

When a new device connects:
1. websocket.accept() — formally accepts the incoming connection
2. Stores the connection in the dictionary under the device's ID
3. Immediately broadcasts a status message to ALL connected devices announcing who just joined

----------------------------------------

disconnect() method
python
def disconnect(self, device_id: str):
    self.active_connections.pop(device_id, None)

Removes the device from the dictionary when it disconnects. pop(key, None) safely removes the key without raising an error if it does not exist.

----------------------------------------

send_to_device() method
python
async def send_to_device(self, device_id: str, message: dict):
    ws = self.active_connections.get(device_id)
    if ws:
        await ws.send_text(json.dumps(message))

Looks up a specific device in the dictionary and sends it a message. json.dumps() converts the Python dictionary to a JSON string before sending.

----------------------------------------

broadcast() method
python
async def broadcast(self, message: dict):
    disconnected = []
    for device_id, ws in self.active_connections.items():
        try:
            await ws.send_text(json.dumps(message))
        except Exception:
            disconnected.append(device_id)
    for device_id in disconnected:
        self.disconnect(device_id)

Sends a message to every device in the dictionary. If sending to any device fails (because it silently disconnected), it is added to a cleanup list and removed afterwards. This prevents a broken connection from crashing the entire broadcast.

----------------------------------------

WebSocket Endpoint
python
@app.websocket("/ws/{device_id}")
async def websocket_endpoint(websocket: WebSocket, device_id: str):
    await manager.connect(websocket, device_id)
    try:
        while True:
            data = await websocket.receive_text()
            message = json.loads(data)
            message["from"] = device_id
            message["timestamp"] = datetime.now(timezone.utc).isoformat()

            target = message.get("target", "all")
            if target == "all":
                await manager.broadcast(message)
            else:
                await manager.send_to_device(target, message)
                await manager.send_to_device(device_id, message)
    except WebSocketDisconnect:
        manager.disconnect(device_id)
        await manager.broadcast({ "type": "status", "event": "device_disconnected", ... })

This is the URL that all clients connect to. The {device_id} part is dynamic — whatever name is in the URL becomes the device's ID.

Step by step:
1. Device connects → manager.connect() is called
2. while True loop keeps the connection alive indefinitely
3. Every incoming message is received as text and parsed from JSON
4. The server stamps the message with from (who sent it) and a timestamp
5. If target is "all" → broadcast to everyone
6. If target is a specific device ID → send only to that device, and echo back to the sender for logging
7. If the client disconnects → WebSocketDisconnect exception is caught, the device is removed, and everyone is notified

4. device_client.py — Line by Line

Configuration
python
DEVICE_ID = "rpi-device-01"
SERVER_URL = f"ws://localhost:8000/ws/{DEVICE_ID}"

Change DEVICE_ID to a unique name for each physical device before deploying.

----------------------------------------

execute_action()
python
def execute_action(action: str) -> str:
    handlers = {
        "laser_trip": lambda: "Laser trip system activated.",
        "system_lock": lambda: "System locked.",
        ...
    }
    handler = handlers.get(action)
    if handler:
        return handler()
    return f"Unknown action: {action}"

A dictionary maps action names to functions. In a real deployment, replace the lambda functions with actual hardware control code, for example GPIO pin commands for the Raspberry Pi.

----------------------------------------

run()
python
async def run():
    async with websockets.connect(SERVER_URL) as websocket:
        async for raw in websocket:
            msg = json.loads(raw)
            if msg.get("type") == "command":
                target = msg.get("target", "all")
                if target not in ("all", DEVICE_ID):
                    continue
                action = msg.get("action", "")
                result = execute_action(action)
                await send_response(websocket, action, result, "co-dashboard")

Connects to the server and listens in a loop. When a command arrives, it checks whether the command is meant for this device. If yes, it executes the action and sends a response back.

Fires when the connection is successfully established. Updates the status label to "Connected".

Fires every time the server sends a message. Prints the raw JSON to the log textarea.

ws.onclose
Fires when the connection is closed. Updates the status label to "Disconnected".

javascript
ws.send(msg);

Sends the text from the message input box to the server as a raw string.

6. Message Formats

Command Message (sent by CO to devices)
json
{
    "type": "command",
    "action": "laser_trip",
    "target": "all",
    "timestamp": "2026-03-18T12:00:00.000Z"
}


Response Message (sent by device back to CO)
json
{
    "type": "response",
    "action": "laser_trip",
    "status": "ok",
    "message": "Laser trip system activated.",
    "from": "rpi-device-01",
    "target": "co-dashboard",
    "timestamp": "2026-03-18T12:00:01.000Z"
}


Status Message (auto-generated by server)
json
{
    "type": "status",
    "event": "device_connected",
    "device_id": "rpi-device-01",
    "connected_devices": ["co-dashboard", "rpi-device-01"],
    "timestamp": "2026-03-18T12:00:00.500Z"
}

7. Full Communication Flow


1. CO opens browser → connects to ws://localhost:8000/ws/co-dashboard
2. Server adds co-dashboard to active_connections
3. Server broadcasts device_connected status

4. Raspberry Pi runs device_client.py → connects to ws://localhost:8000/ws/rpi-device-01
5. Server adds rpi-device-01 to active_connections
6. Server broadcasts device_connected status to all (co-dashboard receives it)

7. CO sends: { "type": "command", "action": "laser_trip", "target": "all" }
8. Server receives message, stamps from and timestamp
9. Server sees target is "all" → broadcasts to every connected device
10. rpi-device-01 receives the command
11. Device checks target — matches "all" so it proceeds
12. Device calls execute_action("laser_trip") → returns "Laser trip system activated."
13. Device sends response back to server targeting "co-dashboard"
14. Server broadcasts the response
15. CO dashboard receives the response and logs it

9. Supported Actions

| Action       | Description                     |
|--------------|---------------------------------|
| laser_trip   | Activates the laser trip system |
| system_lock  | Locks the target system         |
| alert_all    | Triggers an alert broadcast     |
| shutdown     | Initiates a shutdown sequence   |
| ping         | Health check, returns pong      |
| reset        | Resets the target device        |

To add a new action, extend the handlers dictionary in device_client.py:
python
"my_new_action": lambda: "My action executed."
